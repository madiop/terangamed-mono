Je veux que tu conçoives et implémentes une application complète de gestion de cabinet médical avec une architecture microservices moderne, scalable et prête pour la production.

Organisation du projet :
- Backend dans un dossier : terangamed-backend
- Frontend dans un dossier : terangamed-frontend

---

Stack technique imposée :

Backend :
- Spring Boot + Spring Cloud
- Accès aux données : Spring Data JPA avec utilisation OBLIGATOIRE de Specifications
- Base de données : PostgreSQL
- Sécurité : Keycloak (OAuth2 / OpenID Connect)
- API documentation : Swagger / OpenAPI (springdoc-openapi)
- Communication inter-service : OpenFeign ou WebClient
- Résilience : Resilience4j

Frontend :
- Angular
- Authentification Keycloak

Containerisation :
- Docker + Docker Compose

---

IMPORTANT (OBLIGATOIRE) :

- NE PAS utiliser de requêtes JPQL avec (:param IS NULL OR ...)
- Utiliser exclusivement Spring Data JPA Specifications pour les filtres dynamiques
- Tous les endpoints doivent être testables via Swagger avec authentification Keycloak
- Implémenter des tests unitaires et d’intégration avec une couverture minimale de 80%

---

OBJECTIF GLOBAL :

Créer un système où :
- tous les microservices sont indépendants
- toutes les APIs sont sécurisées avec Keycloak
- Swagger permet de tester tous les endpoints avec token JWT
- le système est robuste, maintenable et prêt pour la production

1. Architecture microservices

Inclure :

- API Gateway (Spring Cloud Gateway)
- Service Discovery (Eureka)
- Config Server
- Keycloak (serveur d’authentification)

Microservices métier :

- patient-service
- doctor-service
- appointment-service
- medical-record-service
- billing-service
- notification-service

Chaque service doit :
- avoir sa propre base de données
- être totalement indépendant

---

2. Backend (Spring Boot)

Pour chaque microservice :

- architecture propre :
  - controller
  - service
  - repository
  - dto
  - mapper
- utilisation obligatoire de Spring Data JPA Specifications
- implémentation de filtres dynamiques via Specification
- pagination + tri avec validation des champs (whitelist)
- gestion globale des exceptions
- validation des entrées (Bean Validation)
- logs structurés

Communication inter-services :
- OpenFeign ou WebClient
- gestion timeout + retry
- circuit breaker (Resilience4j)

---

3. Sécurité avec Keycloak

Configurer :

- un realm
- clients :
  - frontend Angular
  - gateway
  - microservices
- rôles :
  - ADMIN
  - DOCTOR
  - RECEPTIONIST

Implémenter :

- sécurisation des endpoints via Spring Security
- propagation du JWT entre microservices
- extraction des rôles dans le backend

IMPORTANT :
- Swagger doit être configuré avec OAuth2 Keycloak
- possibilité de tester les endpoints directement depuis Swagger avec token

---

4. Swagger / OpenAPI

Configurer :

- Swagger pour chaque microservice
- support OAuth2 avec Keycloak
- documentation claire

Objectif :
Tester tous les endpoints via Swagger avec authentification.
